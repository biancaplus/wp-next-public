# Configuration Overview

wp-next is configured through a combination of:

1. **Environment variables** — WordPress URLs and secrets
2. **CLI flags** — command-specific options
3. **`createWpClient` / `createCachedWpClient` options** — runtime data and caching behavior

## Current Configuration

Generated projects currently read WordPress connection settings from environment variables and the generated `lib/wp-client.ts`:

```ts
import { createWpClient } from '@wp-next-public/core';

export const client = createWpClient({
  baseUrl: process.env.WP_URL || 'https://your-wp-site.com',
  mode: 'rest',
});
```

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `WP_URL` | Yes | WordPress site base URL |
| `WP_HMAC_SECRET` | For HMAC | HMAC shared secret |
| `REVALIDATE_SECRET` | For webhooks | Webhook secret for ISR |

## Runtime Caching

See [Caching](/config/caching) for `createCachedWpClient` configuration.

## Roadmap

`wp-next.config.ts` and the `WPNextConfig` type are planned for a future configuration layer. They are not generated by `wp-next init` today.
